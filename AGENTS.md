# Codex Working Rules

## Plan before action

For every task:

1. First inspect the relevant files and understand the existing implementation.
2. Do not modify files, generate code, run commands, install packages, create files, delete files, or change configuration yet.
3. Present a concise implementation plan that includes:

   * your understanding of the task;
   * files likely to be affected;
   * proposed changes;
   * important design decisions;
   * possible risks or uncertainties;
   * tests or validation that should be performed.
4. Stop after presenting the plan.
5. Wait for my explicit approval before making any changes or running any command that can alter the project.

Only proceed when I clearly say something such as:

* “Approved”
* “Proceed”
* “Implement the plan”

If I request changes to the plan, revise the plan and wait for approval again.

## Execution after approval

After approval:

1. Implement only the approved scope.
2. Do not add unrelated improvements or refactoring.
3. Ask before making a significant deviation from the approved plan.
4. After implementation, summarize:

   * files changed;
   * changes made;
   * commands or tests run;
   * remaining issues or recommended follow-up work.

Explicit approval is required for each new task. Approval of one plan does not authorize future tasks.
